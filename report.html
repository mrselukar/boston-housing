Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

when plotted in space look like a grid, hence the name.

Answer: The k-fold cross-validation training technique

k-fold cross-validation training technique is when we do not throw away the training data.
To do so the data is divided into k buckets. K different models are trained with kth bucket as the testing data.
Finally, the average of the k models is taken to get one single model.
As mentioned this technique utilizes all the data as training and testing without violating the rule that same data was not used for both training and testing. This provides advantages over Train/Test split model in which a fraction say 'f' of the data is used as training and '1-f' is used for testing. Increasing f gives better learning result and decreasing improves the validation we want to maximize both, so we use k-fold CV.

K-fold CV for grid search when optimizing a model

As mentioned in Q7 grid search is a technique used for hyperparameter optimization. Using the K-fold CV with grid search, we can use the average testing accuracy as a benchmark to find the best hyperparameters for the algorithm. By using k-fold CV we make sure that the hyperparameters are good for generalizing the data and do not result in a high variance situation.

Drawbacks of k-fold CV

K fold cross validation trains k different models which makes is computationally expensive

Edit Division of data

Learning and testing the model on same data is a mistake as it will result in overfitting.
To avoid this, we use test/train split that is split the available data into training and testing sets and do the training on the training set and the testing on the testing set.
While evaluating hyperparameters using just Test and train split of the data there is still a risk of overfitting on the test set as the parameters can be tweaked until the estimator performs optimally.
To avoid this we divide the data further into a test,train and a cross validation set and training proceeds on the training set, after which evaluation is done on the validation set, and when the experiment seems to be successful, final evaluation can be done on the test set.
The problem now is that there might not be enough data in any one of the set or all the set to make any conclusive decisions. CV is used in this scenario and the data is still spilt into test and train sets but the CV set is not required.
The test sets data is divided into k buckets.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.grid_search import GridSearchCV
#from sklearn.model_selection import GridSearchCV
from sklearn.metrics import make_scorer
def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # Splits the data in testing and training sets randomly 10 times 
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor(random_state=0)

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = dict(max_depth=range(1,11))

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor,params,cv=cv_sets,scoring=scoring_fnc)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
